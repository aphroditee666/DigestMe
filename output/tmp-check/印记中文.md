 长按识别二维码查看原文 https://nodejs.org/en/blog/release/v26.1.0 
 FFI（Foreign Function Interface，外部函数接口）是一种直接调用其他语言（最常见的是暴露 C ABI 的语言）编写的库的机制。 
 Node 多年来在 FFI 方面一直比较复杂。 node-ffi 是早期方案，但相当脆弱。 Koffi 提出了更稳健的方案并持续良好运营，但没有被采纳为官方、核心方案。 
 Node 最终选择了 Node-API / NAPI 方法——编写胶水代码来桥接 Node 和 C++ 的差距，这种方法现在是所有带原生依赖的 Node 库的标准做法。 
 随着 Node 26.1 在 --experimental-ffi 标志下集成了这项工作，我们似乎终于要获得一个核心的、官方支持的 FFI 解决方案了。它还与 Node 的权限模型集成，而且 速度很快 。 
 长按识别二维码查看原文 https://github.com/nodejs/node/pull/63140 
 import { dlopen } from 'node:ffi' ; 
 const path = '/usr/lib/libSystem.B.dylib' ; 
 const { functions } = dlopen (path, { 
 sqrt : { parameters : [ 'f64' ], result : 'f64' }, 
 }); 
 console. log (functions. sqrt ( 2 )); // 1.414213...
