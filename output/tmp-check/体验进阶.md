 怎样的颜色才方便阅读？ 
 Web Content Accessibility Guideline（网页内容无障碍指南，简称WCAG）是由万维网联盟（W3C）下属的无障碍网页倡议组织（WAI）发布的国际网页无障碍标准，旨在通过规范网页内容的设计与开发，确保所有用户（包括残障人士、老年人、色盲用户等）都能平等访问网页信息。 
 WCAG对色彩对比的要求基于文本类型（普通文本/大文本）和 conformance level（ 合规等级 ），分为AA级（最低推荐标准）和AAA级（增强标准）两个层级。 
 底线：普通文本≥4.5:1（AA级），大文本≥3:1（AA级） ——这是全球公认的“最低可读”门槛； 
 理想：普通文本≥7:1（AAA级），大文本≥4.5:1（AAA级）——兼顾舒适性与包容性； 
 WCAG采用相对亮度公式计算文本与背景的对比度，具体步骤如下： 
 将文本颜色（前景色）和背景颜色的RGB值转换为sRGB空间（即归一化到0-1范围）； 
 对每个颜色通道（R/G/B），若其值≤0.03928，则除以12.92；否则按公式((value + 0.055) / 1.055) ^ 2.4计算； 
 计算相对亮度（L）：L = 0.2126*R + 0.7152*G + 0.0722*B（其中R/G/B为步骤2的结果）； 
 计算对比度比值：(L1 + 0.05) / (L2 + 0.05)（L1为较亮颜色的相对亮度，L2为较暗颜色的相对亮度）。

 今日头条的主题色是红色，但它的文本链接有些是红色，有些又是亮蓝色。 
 总结一下规律，用蓝色的文本链接，主要是不想引起注意的地方。所以，我才头条设计师是觉得有些文本链接用红色太过抢眼，所以才用亮蓝色的。 
 那为什么不像微信那样用灰蓝色呢？看看下图对比就知道了： 
 因为页面上的主题红已经很显眼了，文本链接就算用亮蓝色，也不会喧宾夺主。 
 反倒是如果改用灰蓝色，会让文本链接太不起眼，看起来还有点像是无效链接。 
 总结一下 
 当主题色相对于背景色的对比度较低时，可能就不适合给文本用了，具体可以用在线工具来测试。 
 主题色为红色、橙色，有警示含义时，也可能不适合作为文本链接。 
 如果主题色不适合给文本用，那么千万不要勉强用，也没必要为此改主题色。

 数据验证确实是个好东西，但是很多设计师在毫无数据分析经验的情况下，就胡乱编造数据。结果炫技不成，反而给人留下“不诚实”的印象，就糟糕了。 
 其实也不怪设计师们，因为数据分析这东西， 确实没太多适合设计师学的途径 。不论是大学的设计专业还是市场上的设计培训班，都极少有针对设计师的数据分析课。 
 设计师想要提升数据分析能力，推荐来我们的《数据分析训练营》，已经第5期了，每期都大获好评。不论是零基础还是有点儿经验的，都表示受益匪浅，并且立马就能用起来。 
 接下来，我就分析一下，为什么会员转化率提升3%这个数据验证缺乏说服力。 
 1. 价值过高显得不真实 
 转化率提升3%没什么奇怪的，但让会员转化率提升3%，就有点惊人了。 
 要知道，连知乎这种平台，一年都能卖4亿会员费。假设一个设计师能把知乎的会员转化率提升3%，那就意味着，这名设计师凭一己之力帮企业多赚了1200万！ 
 这就好比一个年入千万的高富帅/白富美说看上你了，理智一点的人都会想：“这是不是遇到诈骗了？” 
 2. 缺少严谨性声明 
 假设这个产品的一年的会员费只有100万，会员转化率上涨3%也不过多赚了3万块钱，这么看就很正常了。 
 但要确定这个数据是设计的功劳，是不是得排除营销的影响？好歹得说一句，数据测量期间没有任何营销活动的变动。 
 对方也知道这可能是一句谎言，但加上这么一句声明，至少说明设计师知道怎样的数据才算是严谨的，并且愿意为数据的严谨性负责。 
 3. 缺少方案关联性 
 假设前面的2点都做到了，但这方案的哪个部分，导致了如此漂亮的数据上涨呢？ 
 假设改的地方是「连续包月」，那应该用「连续包月」的转化率来验证，才更能看出效果。 
 假设改的地方是权益展示，那应该用各项权益的点击率来验证，才更能看出效果。 
 总而言之，具体改了哪里，就应该看哪里的数据。 
 如果不管三七二十一，全部只看总数据，就感觉没有那么可信。 
 设计师的数据思维 
 其实真正懂数据的设计师，一定是拿到需求的时候，就开始分析数据了。然后在设计方案的过程中，会围绕着如何提升这些数据进行思考。 
 如果拿到需求的时候不看数据，反而等方案完成了再突然来个数据验证，是很不现实的。 
 因为设计师最有理由申请数据权限的时机，就是刚接到需求的那一刻。反倒是把方案交付完成后，设计师不知道方案何时上线，手上又有别的需求要忙，申请数据权限没那么方便了。 
 所以设计师做数据分析，跟产品经理、市场营销等业务岗位，是完全不一样的。包括如何获取数据、如何整理数据、如何分析出设计策略、如何验证方案……都是不同的。

 4. 有些组件的圆角尺寸不能随便改

 1. 同级别元素的圆角尽量统一

 不管是做电商、社交还是 B 端工具，大家在聊到项目成果时，几乎整齐划一地都会甩出一句：“经过设计改版，核心按钮的点击率（CTR）提升了 XX%。” 
 说实话，第一次看到这种描述，我会觉得这位同学有数据意识；但当我看过几十份甚至上百份雷同的描述后，我产生了深深的怀疑。 
 因为对真正长期做数据分析的人都知道，点击率是一个很“鸡肋“的数据，几乎很少人会在重要汇报上提及。 
 接下来，我就给大家分析一下，不建议分析“点击率”的原因，以及我 推荐 设计师分析哪些数据。 
 点击率有什么问题？ 
 设计师都希望自己设计出来的关键按钮被用户点击，所以会直觉性地认为“点击率”是个很重要的数值。 
 但问题是，有些点击其实根本没有意义。 
 1. 点击可能被重复计算 
 网络/系统卡顿的时候，很多人不耐烦，就喜欢反复点击——这些也都算在点击率里哦。 
 2. 有些点击是无效的 
 很多按钮，没打成某些条件的话，即便可以点击，也是没意义的。 
 例如来到登录页的用户，连手机号都没填完，就点击下一步。点完之后呢？出现一个错误提示，打回去继续填表单。 
 3. 转化率才是关键，跟点击无关 
 转化率的计算，用得不是点击数量，而是访问人数： 
 转化率 = 这一步访问人数 ÷ 下一步 访问人数 
 例如：注册转化率 = 访问注册页的用户数量 ÷ 注册成功页的用户数量 
 因为只有访问人数，才能有效排除重复/无效点击，了解真正的用户行为轨迹。 
 如果你在作品集里混淆了这两者，面试官一眼就能看出你对数据和业务的认知都比较浅显，进而你展示的数据分析也令人怀疑。 
 其实数据分析一点也不难，难的是很多设计师没有机会接触数据和业务。甚至陷入了不懂数据就没机会接触数据，没机会接触数据就更不懂数据的死循环……

 接下来才是重头戏，我给大家列一些设计师应该关注的数据指标（除了转化率之外）。 
 建议设计师关注的数据指标 
 1. 留存率（Retention） 
 点击率解决的是“引诱”问题，而留存率解决的是“留人”问题。 
 这个数据是导入期和成长期的产品必看的，成熟期和衰退期就不太看了。
