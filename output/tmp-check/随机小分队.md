 Shopify 在 2025 年底基本实现了全员 100% 的 AI 工具采用率，而且是在没有任何强制命令的情况下做到的。因为模型本身好用到那个程度，员工自然就用起来了。

 他带来了 Shopify 正在发生的五个真实案例：为何让更多 Agent 并行运行，反而会适得其反？Auto Research 如何用 400 次实验换来 1 次真实突破？一套花了近一年时间校准的"虚拟买家"系统，怎样在没有 A/B 测试数据的情况下预测转化率？以及一个大多数人还没听说过的 AI 模型，正在 Shopify 内部悄悄替代更主流的选择。 
 希望能为大家带来启发，Enjoy！

 Shopify 是全球最大的电商基础设施平台之一，为数百万商家提供建站、支付、物流等服务。它不是一家 AI 公司，但在 AI 工具的内部落地上，走得却比大多数 AI native 公司更快更彻底：全员采用、无上限 Token、自研工具链，并且从零开始搭建了一套完整的 AI-Native 工程体系。 
 一、从"允许用"到"不用就没法干活" 
 1. 工具好用到一定程度，采用率会自己跑到 100% 
 Shopify 一直有一张追踪图，记录每天主动用过至少一种 AI 工具的员工占比。在 2025 年 12 月之前，这个数字增长得十分缓慢。但在 12 月之后，曲线突然变得陡峭，一路攀升到了接近 100%。 
 这是因为过去几年的各项微小改进在此时全部到位，累积形成了质变。 当工具好用到一定程度时，不用它反而会影响工作效率，员工自然就开始主动使用了 。 
 在普及率提升的同时，内部工具的选择也在发生迁移。 
 Cursor 和 GitHub Copilot 这类集成在 IDE 里的工具，用量虽然没有下滑，但增速正在放缓。真正快速增长的是那些不需要打开 IDE 的命令行工具。其中，Claude Code 和 Codex 是增速最快的一批。此外，Shopify 还自研了一款专门为 Coding 设计的 Agent，取名为 River。 
 2. Token 不设上限，且要使用最好的模型 
 他们内部目前推行一套明确的准则： 工具可以自由挑选，Token 也可以不受限制地使用，但要选择最好的大模型 。管理层不会主动干预个人的使用量，但如果有人为了省钱而改用小模型，则会被建议换回。 
 从实际情况上来看， Token 消耗的分布正变得越来越不均匀 。高频使用者的增长速度远快于其他员工，长期来看，少数几个人可能会占用大部分配额。但是这种情况，并不急于采取限制措施。最好是等待内部经验逐步扩散，带动更多人深度使用，到时候消费分布自然会趋于平稳。 
 二、Token 经济学：烧多少不是关键，烧在哪里才是 
 1. Token 是思考时间，但烧在哪里比烧多少更重要 
 英伟达 CEO 黄仁勋（Jensen Huang）曾公开说：如果一名年薪 50 万美元的工程师每年在 Token 上的花费不足 5,000 美元，那其实是在浪费资源。可能有人会觉得这是“自卖自夸”，但大方向是正确的。 
 在现阶段， Token 消耗量代表的就是思考时间 。一名工程师如果舍不得花时间思考，很难产出好结果；同样，如果一个团队舍不得让模型消耗 Token，产品的质量上限也会受到限制。 
 但这并不意味着 Token 用得越多越好， 消耗的总量从来不是核心，钱花在哪里才是关键 。 
 2. 最容易踩的坑：让 Agent 并行跑但不让它们互通 
 他们在实践中发现了一个明显的误区： 同时启动太多相互不通信的并行 Agent 。 
 直觉上，多个 Agent 同时工作看起来是在"充分利用算力"。但实际结果往往比不上减少 Agent 数量，加上精心设计的 Critique loop（批判循环） 。也就是让一个 Agent 先生成代码，再让另一个 Agent（最好换一个不同的模型）来批评它、提出改进建议，第一个 Agent 再根据反馈重新修改，如此循环。 
 这种机制的代价是延迟更高，因为用户需要等待模型之间完成“辩论”。但代码质量会有显著提升。 
 "人们虽然不喜欢等待，但就在等待的过程中，代码已经变得更好了。" 
 3. 真正值得追踪的指标 
 Token 的总消耗量意义有限。真正值得关注的是一个比值： 代码生成阶段消耗的 Token 与 PR review 阶段消耗的高质量模型 Token 之间的比例 。 
 在 PR review 阶段，不能使用小众模型敷衍了事。为了真正拦截生产环境中的 bug，需要动用 GPT-5.5 或 Gemini 3.1 这样的旗舰模型，让它们轮流进行深度审查。虽然这样做成本很高，但这笔钱花得非常值得。一小时的深度审查，往往可以省去后续三天的排查、回滚和重新测试工作。 
 三、代码量爆炸，但总 Bug 数也跟着涨了 
 1. 代码产量激增，总 Bug 数不降反升 
 在 Shopify 内部，PR 合并数量的月环比增速已经从 10% 提升到了 30%，代码复杂度也在同步提高。这是生产力提升的信号，但也带来了前所未有的压力。 
 从数据来看，AI 编写的代码其实更“稳”，平均每行代码出错的概率比人类工程师更低。但问题在于，AI 产出的代码量实在是太大了。 庞大的基数使得 Bug 的绝对数量反而有所上升 ，单行质量的改进被总量的爆发彻底淹没了。 
 应对方式只有一个：在 PR review 这道关口投入更多资源。 用最好的模型，让模型之间轮流评审，花更长时间，但把 Bug 拦在测试环境里 。从而省掉后续排查失败、回滚部署所消耗的总成本。 
 因此，不要只关注单个 PR 的处理速度，而要关注 整个部署周期的总时长 。在这种逻辑下，一次昂贵、细致的 Review，其实比追求快速通过更合算。 
 2. PR 数量暴涨，整个部署周期反而变慢了 
 当 PR 数量呈爆发式增长时，整个部署周期反而慢了下来。在过去，人类工程师的产出速度让 PR 机制表现尚可，人工解决冲突也还算及时。但 AI coding 已经进入了机器速度。 
 由于 PR 数量巨大，每一个 PR 都在排队等待 CI/CD 管道完成检测。这导致测试时间不断拉长，失败率也随之升高。虽然 Shopify 目前正在使用 Graphite 的 Stacked diffs 来缓解压力，但这也只是暂时的。 
 现有的 CI/CD 设计逻辑完全是围绕人类的工作节奏建立的，在 AI 时代，这套流程需要一次彻底的重塑。如果能让代码以微服务的方式独立部署，管理这种拆分后的复杂度反而变得更容易。 
 微服务架构可能要重新回到主流视野了 。 
 四、Shopify 的自动化实验：Tangle 与 Tangent 的应用实践 
 1. 实验跨团队自动共享，无需额外沟通 
 每一个做过机器学习或数据科学的人可能都有过这种经历：6 个月前好不容易跑通了一个实验，今天想重跑时，却发现路径变了，脚本也崩了。甚至连特征命名都对不上，只能花上 1 个月的时间在旧代码堆里翻找，就像是在做“数字考古”。 
 当这种问题发生在整个团队或多个部门时，每天都会浪费大量的工程时间，对于一个刚入职需要 Onboarding 的新人来说更是如此。 
 Tangle 是 Shopify 内部开发的第三代数据处理与 ML 实验平台。它的核心设计理念是基于内容“Hash”的缓存机制。 
 简单来说，就是 只要数据处理任务的输出没有变化，系统就不会重新运行 ，无论代码如何更新或路径如何改变。 
 当多个团队独立启动需要相同预处理步骤的实验时，系统会自动检测到重复。任务只会执行 1 次，后启动的团队会发现那些步骤被“自动跳过”了，他们可能都不知道是另一个团队已经帮他们完成了这些工作。 
 这种效果被称为“人与人之间的网络效应”：不需要任何沟通成本，整个组织层面的计算效率却能自动累加。 
 Tangle 的另一个关键设计是“开发即生产”。实验代码不需要移植到另一套生产系统，实验跑完后可以直接一键部署，使用的也是同一份代码和同一个环境。那种“开发环境正常、生产环境出错”的经典难题，从源头上就被消除了。 
 2. 400 次实验仅有 1 次成功，带来根本性突破 
 Tangent 是构建在 Tangle 之上的自动研究循环。它像是一个 Agent，会持续运行实验、分析结果并修改参数，不断优化目标函数，直到达成预设目标或资源耗尽。 
 效果比预想的还要直观。 
 Shopify 的搜索服务 QPS 从 800 提升到了 4,200，在保证相同质量的前提下，吞吐量提升了 5 倍以上。这一提升并没有增加任何机器设备，也没有改变系统架构，完全来自于 Tangent 自动研究循环的不断分析、修改与测试。 
 除此之外，Tangent 还自动识别出了一张冗余的数据表。那是当时最大的一张数据表，作用仅仅是将一个随机 ID 映射到另一个随机 ID。Agent 发现并清理了这个没有意义的存储。 
 Mikhail 还做了一个私人测试，想看看自己已经充分优化的个人项目是否还有提升空间。他让 Tangent 在生产规模的配置下跑了几周，总共做了 400 次实验。其中只有 1 次找到了真实的改进，成功率仅为 0.25%。 
 但那个改进确实有效，而且是他自己多年来都没能发现的。如果由人来手动做 400 次实验，命中率或许更高， 但没人会真的花 3 年时间去做这件事 。对机器来说，成本只是电费，却能找到人类无法发现的东西。 
 3. 用量最高的不是工程师，是一名 PM 
 比较意外的是，Shopify 内部 Tangent 使用量最高的，并不是工程师，而是一名产品经理。 
 这是 Auto research 真正普及的标志。产品经理拥有行业知识，清楚什么样的结果才算好，也明白产品的目标在哪里。他们不需要了解复杂的算法细节，那一层由 Tangent 来处理。就像 Claude Code 降低了写代码的门槛一样，Tangent 也降低了训练和优化模型的门槛。 
 以前很多人批评 AutoML 没什么用。在 LLM 出现之前，这个判断确实没错，因为当时它只能在狭窄的领域内勉强发挥作用。但 LLM 提供了通用推理能力，将 AutoML 从规则框架内的参数优化，转变为真正理解问题并创造新的解法。这是质的变化，而不只是量的累加。 
 五、Shopify 的 AI 买家模拟：从商业实验到反事实推演 
 1. 没有真实数据支撑，模拟出来的只是你想要的答案 
 在 Shopify 内部推进 “客户模拟” 这个想法时，创始人 Tobi Lütke 最初也提出过质疑：如果只是通过 Prompt 让 Agent 扮演一个买家，那它不就是按照指令在演戏吗？这样做到底有什么实际意义？ 
 对于没有历史数据积累的公司来说，这个批评切中要害。但 Shopify 的优势在于拥有几十年的商家经营数据。这些数据详细记录了哪些页面的改动能提高转化率，哪些会导致销售下滑，以及在什么时段、针对什么客户群，不同品类的表现有何差异。 
 这些真实数据成了校准虚拟买家行为的基础 。Sidekick Sim 系统模拟出来的 Agent，行为模式锚定的是真实消费者在几十年里积累下的历史规律。它们是跟着数据走，而不仅仅是跟着 Prompt 走。 
 Shopify 团队花了将近 1 年时间来反复校准这套系统。内部设定的目标是：模拟行为与真实 A/B 测试中“加入购物车”数据的相关系数要达到 0.7。 
 相关系数达到 0.7，意味着当 Sidekick Sim 预测“版本 A 比版本 B 转化率更高”时，真实实验的结果与它高度一致。为了让这个数字从 0 爬升到 0.7，团队投入了近 1 年的精力，每一次细微的提升，都意味着更精细的数据清洗和更严格的 Agent 行为校准。 
 2. 弥补中小商家无法进行 A/B 测试的空白 
 对于大型商家来说，AB 测试是日常工具。但 Shopify 平台上绝大多数是中小商家：一周的流量不足以产生统计显著的样本量，跑不了正经的 AB 实验。他们面对的真实困境，往往连"A 和 B 应该是什么"都还没有答案，更谈不上比较哪个更好。 
 Sidekick Sim 为这些商家做的是：分析他们的店铺主题、页面布局和图片，通过运行模拟，直接告知商家哪些设计可能会降低转化率，并给出修改建议。 
 实验中有一个反直觉的发现： 把图片放大通常反而会降低转化率 。 
 由于从 HTML 代码层面很难观察到图片大小对视觉产生的具体影响，Sidekick Sim 必须启动真实的 Headless browser 进行页面渲染，再利用多模态模型分析视觉效果，才能精准检测到这种细微的差异。 
 这也解释了为什么这套系统这么贵。大规模运行 Headless browser 加上旗舰级多模态模型，带来了巨大的开销。目前团队面临的主要挑战是如何处理高昂的账单，以及如何通过模型蒸馏等技术降低成本，来应对不断增长的流量需求。 
 3. 从模拟买家到对整个商家进行推演 
 在 Sidekick Sim 之下，Shopify 还有一个更庞大的内部系统：基于 HSM（层次结构模型）的商家行为图谱。 
 它把整个商家的经营行为建模为一个"实体在时间轴上的行动轨迹"。在这张图里，任何一个节点都可以被"干预"。比如给这位顾客发一张优惠券会怎样，不发会怎样，在第 3 次浏览时发和在第 7 次浏览时发有何区别？并向前推演出两条不同的行为轨迹，并且比较最终的结果。 
 这是 反事实推演（Counterfactual analysis） 在真实商业场景中的落地：不依赖 AB 测试的随机对照组，而是直接在同一条时间线上做"如果当时不一样"的推演。 
 他们用这套能力做了一件具体的事：自动识别哪些商家的特定销售环节出了问题，并且主动通知他们："你的加拿大区域配置看起来有误，这是建议修改方案"，或者"你在这个客户群上的复购激活还有空间，建议在第 7 天发一封这样的邮件"。 
 这种对复杂人类行为系统进行建模并完成反事实优化的设想，在以前几乎是不可能实现的。但有了 LLM 之后，它突然从一种理论设想变成了可以落地的工程任务。 
 六、Liquid AI：在生产环境中落地的非 Transformer 架构 
 1. Liquid AI：成功打入生产环境的架构例外 
 如果要推荐一种模型架构， Liquid AI 的液态神经网络是目前的最佳选择。它是目前已知唯一在生产环境中真正具备竞争力的非 Transformer 架构 。 
 理解这一点需要了解一些技术背景。在 Transformer 架构之外，状态空间模型（SSM）一直是另一条备受关注的技术路线。SSM 的优势非常明显：它处理长上下文时的计算复杂度是线性的，而不像 Transformer 那样呈二次方增长。 
 这意味着它在处理长文本时效率更高。虽然一直有初创公司在研究 SSM，但它始终没能真正进入主流视野，原因在于表达能力不足。在需要复杂推理的任务中，SSM 的表现上限明显低于 Transformer。 
 Liquid Neural Networks 是 SSM 的进化版本。它基于微分方程推导而来，计算展开后相当于一种特殊的卷积。这种架构不仅保留了处理长上下文的高效率，还拥有更强的表达能力。目前，Liquid 与 Transformer 结合的混合架构被认为是行业内最优的架构组合之一。 
 2. 兼顾极低延迟与超大规模批处理 
 Shopify 利用 Liquid AI 处理两类需求截然不同的场景，而这两类场景恰好都能发挥 Liquid AI 的优势。 
 第一类是 极低延迟需求 。 
 在搜索功能中，用户输入查询词后，系统必须在 30 毫秒内完成查询理解工作，包括同义词扩展、结合个性化历史以及语义分析，并将结果输出给下游服务。这里使用的是一个参数量约为 30,000 万的 Liquid 模型。为了达到 30 毫秒的严苛标准，团队与 Nvidia 以及 Sentinel ML 合作，进行了大量的底层 CUDA 优化。在模型参数较小且要求低延迟的场景下，许多在大模型中不被重视的细微因素都会显著影响性能，因此团队必须进行逐一优化。 
 第二类是 超大规模的批处理需求 。 
 当新商品上架时，系统需要自动完成分类、属性提取以及语义聚类，比如判断不同商家的某件产品是否为同一种商品。这类场景面对的是数十万万量级的商品，虽然对延迟不敏感，但非常看重吞吐量和长上下文处理能力。团队通常将大型基础模型的能力蒸馏到 700,000 万至 800,000 万参数规模的 Liquid 模型中。在处理这类任务时，Liquid 版本的表现往往优于阿里的 Qwen 大模型。 
 3. 成本压力越来越大，蒸馏正在成为标配 
 随着 Token 消耗量的爆炸，模型蒸馏正在变得越来越重要。 
 将大型模型的能力压缩到更小、更专注的模型中 ，从而以远低于原始模型的成本运行特定任务，这种路径在实践中变得越来越频繁。 
 Liquid 模型就是理想的蒸馏目标：架构表达力够，足以承接复杂任务的蒸馏知识；适合在延迟要求严苛或超大吞吐的场景下运行。同时，它处理上下文的效率极高，非常适合需要分析长内容的任务。

## 5. Elad Gil：AI初创要尽早考虑退出，现在是人与AI协作黄金时间！

- 链接: https://mp.weixin.qq.com/s?__biz=MzI0NTAwMzU1MA==&mid=2247505788&idx=1&sn=61c2bd7c1a9bfa61196dfe0777641e7c

随机小助手 2026-04-26 19:59 美国 
 留给人类与AI“共处”的时间可能已经不多了？ 
 Linkloud 引言 
 OpenAI 和 Anthropic 的年化收入双双逼近 300 亿美元，仅这两家公司，就各自吃下了美国 GDP 的 0.1%。AI 从零增长到这个体量，只用了几年时间。 
 硅谷知名投资人 Elad Gil （投资组合包括 Airbnb、Stripe、OpenAI、Perplexity 等）最近写了一篇 AI 行业随想，12 个话题涵盖算力天花板、Token 经济学、隐性裁员、AI 的"闭环吞噬"逻辑，以及一个犀利判断：大多数 AI 公司应该认真考虑在未来 12 到 18 个月内退出。希望能为大家带来启发。Enjoy！ 
 更多完整内容，请点击阅读原文。 
 01. AI 吃掉了多少 GDP？ 
 OpenAI 和 Anthropic 各占美国 GDP 的 0.1%，2030 年 AI 能占到多少？ 
 美国 GDP 大概 30 万亿美元。OpenAI 和 Anthropic 目前的年化营收据传均在 300 亿美元左右，各自约占整体 GDP 的 0.1%。 
 把云服务和其他 AI 相关收入加进来， AI 在短短几年内就从零增长到了美国 GDP 的 0.25% 到 0.5% 。如果两家公司能像许多人预测的那样在今年年底实现千亿美元营收，那么到 2026 年底，AI 的年化贡献将接近 GDP 的 1%。这个速度，说"惊人"并不夸张。 
 2030 年 AI 会占 GDP 的多少？2035 年呢？美国经济体量的天花板会不会反过来拖慢 AI 的实际渗透？生产力增益又有多少会像 2000 年代互联网或 80、90 年代 IT 浪潮那样，消失在 GDP 统计的盲区里？ 
 顺带一提，如果 AI 的影响被系统性低估，围绕它的监管政策可能也会走偏：人们只看到坏的一面，比如岗位流失；却忽略了好的一面，比如新型就业机会和对教育和医疗的改造…… 
 也许真正的 AGI 时刻，恰恰是当 AI 能够精确衡量美国 GDP 和生产力增益的那一天。 
 02. AI research 圈刚完成了一场"分布式 IPO" 
 一家公司上市之后，早期员工往往一夜之间财富大增。这种变化会影响行为，有人开始分心买房、追逐地位、沉溺社交，或者跑去做些和主业无关的"人生副本"。当然不是所有人，但总有一部分人会经历这样的状态漂移。 
 Meta 大手笔抢人，直接搅动了 AI research 人才市场，各大顶尖 AI lab 不得不跟进，大幅拉高研究员的薪酬包。某种程度上，整个 AI research 圈刚刚完成了一次横跨多家公司的"集体 IPO"：从大型 AI lab 到大厂的一个截面，数十乃至数百位顶级研究员，因 Meta 的竞价而在同一时间段内坐收了可观的财富。 
 和传统 IPO 如出一辙，这批人当中，有人开始调整工作节奏，有人悄然离场，也有人依然埋头向前。 
 总体而言，AI 领域的大家对"构建 AGI"或"推动 AI for Science"有很强的使命认同。 
 而且硅谷还悄悄发生了一件新鲜事：不是某家公司上市，而是一批特定的人同步迈入了"后经济时代"。这批顶尖 AI researcher，从此不再为钱工作。（最接近的历史类比，可能是早期加密货币的 HODLR。） 
 03. 算力天花板会不会强化寡头格局？ 
 过去几年，模型能力的进步有目共睹，也直接催生了大量新的应用场景。头部 AI lab 和围绕 AI 搭建的应用公司，收入都在快速增长。 
 但与此同时，随着训练规模和未来推理需求的不断扩张，各大 AI lab 正越来越明显地感受到算力瓶颈。HBM 内存的供给端，例如海力士、三星、美光等，至少在未来 2 年内将持续受制于扩产周期，整体产能难以快速跟上。 
 这意味着，没有哪家 AI lab 能大幅超前囤积算力，也没有谁能随心所欲地用。 
 所有头部玩家都处于日益明显的"算力饥荒"中 。这种约束，可能会在短期内给模型能力的进化画一条人为的渐近线。尽管各家在现有算力上的效率优化仍在持续，但因为这道约束，在 2028 年之前，可能没有任何一家 AI lab 能够拉开决定性的领先优势， LLM 市场的寡头格局反而会被进一步巩固 。 
 同时，还会出现各种连锁反应。各家 AI Lab 在应用层和模型层之间的资源分配会反复摇摆。芯片和系统的折旧周期也将超出此前的预期，因为新产能补不上来，现有硅片的服役寿命会被拉长。 
 当然也存在反面可能：如果某家 Lab 在算法层面取得重大突破（而且这个突破没有在旧金山某个节日派对上被研究员泄漏出去），它就有机会一骑绝尘。 尤其在 AI coding 全面起飞、出现某种 AI 造 AI 的自我改进循环的情况下 。但如果算力始终是硬约束，这种"起飞"可能要等到 2028 年甚至更晚。 
 04. 算力（Token）是新货币 
 算力，也可以说是 Token，正在成为硅谷衡量经济价值的新计量单位 。Token 预算直接决定了三件事： 
 一个工程师能撬动多大的事， 
 一家公司的支出上限和收入天花板， 
 以及它的商业模式。 
 有些公司本质上是披着工具外衣的推理服务商。Neocloud（新型算力云）是最典型的形态，而像 Cursor 这样的产品也在把低价推理当作核心产品能力的一部分，它们实际上是在用补贴算力来拉新和留存。谁不喜欢多一点免费 Token 呢？ 
 有趣的是，做环保运动鞋的 Allbirds 刚宣布卖掉鞋业、更名为 NewBird AI，并融了 5,000 万美元可转换债券买 GPU 来做算力租赁，股价一天暴涨 500% 多。它会成为 AI 领域的 MicroStrategy 吗？（MicroStrategy 是一家把几乎全部资产押注比特币的上市公司，靠囤币而非主业驱动市值。） 
 05. 隐性裁员与发展中国家 
 目前多数"AI 导致裁员"的新闻，大概率只是在偿还零利率时代过度招聘的债。"我们 AI 用得好所以不需要那么多人"，听起来总比"当年招多了，现在纠偏"好听得多。 
 但 AI 确实在部分领域产生了实质影响，客服就是最典型的。那些因 AI 缩减团队的公司，最先砍的往往不是自己人，而是外包商。这些人头不在公司资产负债表上，而是以服务费形式支付。于是印度、菲律宾这些外包服务重镇，反而可能最先承受冲击。 
 更深层的影响在于，如果 AI 率先夺走的是外包服务岗位，这些发展中国家原本赖以实现经济跃升的"服务业阶梯"就可能断掉。就业结构将被迫转型，甚至不排除影响人口的迁移走向。 
 06. 很多公司的人员规模即将"封顶"，然后缩减 
 多位后期阶段的 CEO 告诉 Elad Gil，比起大规模裁员， 他们更倾向于"不再增长" 。哪怕公司收入增速达到 30%、50% 甚至 100%，人头也保持持平甚至略降，通过自然流失消化规模。 
 现有人员的人效会更高，公司也可能 开始用"少量更优秀的人"替换"大量普通的人" 。中期来看，这会推高顶尖人才的身价，尤其是那些能极致利用 AI 杠杆的人。招聘不会停，但会更集中在销售和部分工程岗位，其他方向可能明显收缩。 
 有些公司已经在思考一个新指标：Token 预算和薪资支出的比例应该是多少？目前还没有答案。 
 真正的早期创业公司，比如 5 个人的团队，短期内还是会像以前一样扩人 ，只不过每个人的杠杆更大。 "人头封顶"更多是中后期或上市公司在增长期的现象 ，预计未来 2 到 4 年会越来越普遍。至于低增长的公司，缩编几乎已成了必然。 
 这一趋势对 HR 软件等赛道也有连带影响。 
 07. "Slop 时代"可能是 AI 与人类协作的黄金时代 
 我们大概率正处在 AI 加人类的黄金时代 。 
 几年前，AI 还不够普及，泛化能力弱，只能执行特定任务。未来，AI 可能在大多数任务上超越人类，接管很多人觉得有趣的工作。但眼下这个阶段很特别：AI 能大批量生产"还行但不够精"的内容，也就是 Slop（粗制滥造的产物）。 
 人类仍然被需要，负责给 Slop 去糙、做判断、把关质量，而 Slop 本身又实实在在地提供了时间和产出上的杠杆。 所以现阶段的工作体验反而是最好的 。一旦 AI 全面接管，这个黄金窗口可能就会关闭。 
 换个角度看，在 AI slop 时代之前，我们其实已经处在"人类 slop 时代"了：互联网膨胀到数十亿网页，但并没有产生数十亿条真正有价值的新洞见。这场粗制滥造的 slop 时代，也许会随 AGI 的到来而终结；又或者 AGI 要做的第一件事，是清理掉之前几波人类留下的全部 Slop。 
 08. AI 会先吃掉"闭环" 
 AI 会率先自动化那些容易形成闭环学习系统的任务 。 
 Coding 和 AI research 可能最先被加速，然后被替代，因为它们可以搭建可测试的闭环系统，让机器快速学习和迭代。 闭环越紧，AI 学得越快 。 
 可以画一个 2x2 矩阵：纵轴是形成闭环的难易度，横轴是经济价值。两者兼备的区域，就是 AI 最先冲击的地方。软件工程首当其冲。 
 但 Coding 领域有一点特殊： 目前市场对优秀开发者的需求大概是供给的 10 到 100 倍 ，这也是 AI coding 工具增长如此迅猛的原因。未来的 AI 工程师更多是管理和编排大量 Agent 去构建产品，侧重系统思维和产品思维，而不是自己逐行写代码。 
 下一批被纳入紧密闭环的任务或岗位会是什么？AI 在哪些地方还有更大的嵌入和学习空间？值得持续关注。 
 与此同相关的，各行各业的数据采集和标注的需求也将持续增长。 
 那些深度追求"代码即手艺"、享受精雕细琢的工程师，在 AI 时代可能正变得越来越不适应。反而是系统思维和产品思维更强的工程师如鱼得水。当然，大多数人是两者的混合体。 
 09. Harness：工具链与工作流 
 观察 AI coding 工具的使用现状，Harness 的粘性正在变强。所谓 Harness，就是围绕模型搭建的工具链、产品体验和工作流。 
 用户的选择不仅取决于模型本身，还取决于用户围绕模型构建的环境和提示词策略。 
 品牌效应也比很多人想象的更重要 。最终走向无非两种：要么某个 Coding 模型大幅领先，要么各家持续胶着。Harness 对长期防御力到底有多大贡献，仍是悬而未决的问题。 
 产品往往在某个临界点之前毫无粘性，过了那个点之后又极其难以替代。 
 不同领域中 Harness 的重要性也会分化。销售领域的 AI harness 是什么？AI 架构师的 Harness 又是什么？这些空白地带，为一些创业公司留出了生存空间。 
 10. 卖的是劳动力，不是软件 
 AI 重新定义的，是在线劳动力单位的销售方式（以及未来通过机器人进入实体世界），而不仅仅是替代软件。Zendesk 卖的是客服坐席的授权，而 Decagon 和 Sierra 卖的是 Agent 实际完成的客服工作成果。 
 AI 正在大幅扩展整个科技行业的 TAM。 
 11. 大多数 AI 公司应该认真考虑在未来 12 到 18 个月内退出 
 1995 到 2001 年的互联网时代，大约有 2,000 家公司上市，最终存活下来的不过十几二十家。AI 时代大概率会重演类似的剧本： 大多数公司，包括那些当下收入增长迅猛的，最终都会被市场变化、竞争加剧和采用周期反噬 。

 创始人不再被强制要求具备硬核技术背景，也不再需要头顶斯坦福或麻省理工学院计算机系的耀眼光环。只要在垂直领域拥有足够深厚的专业知识就够了。 
 一个全新的品类由此诞生： Lean AI（精益 AI）公司 ，借用精益创业（Lean Startup）的理念，指那些用极少资本、极高效率实现快速增长的 AI 公司。 
 它们 从成立第一天起就保持着极高的资金使用效率 ，不少甚至在零融资的情况下实现了盈利。过去，传统软件公司 2 倍的年增长率就已经很亮眼，但现在这批 Lean AI 公司的年增长率往往能飙升到 10 倍。实现盈利的速度也超越了以往任何一代软件企业。 
 目前 Yash 关注的项目库里就已经有 8 家 Lean AI 公司达到了 AI 行业前 10% 的毛利水平，同时保持着两年前根本不可能出现的增速。 
 传统的 VC 机构面对这种底层逻辑的转变显得毫无准备，而随着创业门槛不断降低， "创始人"这个群体的定义和体量正在急剧扩大 。 
 02. 被传统 VC 系统性忽略的创始人 
 过去十年，每个 VC 心里都有一个”完美创始人”模板：斯坦福计算机系毕业，或者 MIT 工程师，总之是某种特定模具里出来的”聪明人”。 
 但现实是，各行各业的创始人正在以惊人的规模把公司做起来。他们不符合传统模板，没有过往 VC 通常情况下会看重的背景光环，但他们有一样更值钱的东西： 对特定垂直领域的深度认知 。 
 这批新型创始人有两个根本性的不同。 
 第一， 极度在意股权稀释 。他们见过太多融资过快过多的教训， 追求资金效率不是被逼无奈，而是刻意为之 。能少稀释就少稀释，对其他融资方式也越来越感兴趣。 
 第二， 增长太快、盈利太早 ，传统 VC 的体系反而拖慢他们。传统机构投资人的服务方式天然不适配这类创始人。他们想要的是一个真正下场帮忙的战略伙伴： 帮招人、帮融资、帮引荐客户 ，同时知道什么时候该让开。更关键的是，这个人得真正理解如何用 AI 打造极致精益、快速扩张的公司。 
 03. 怎么评估 90 天增长 10 倍的公司 
 传统 SaaS 投资人习惯等 12 到 18 个月的留存数据才出手，要看几个季度的增长曲线，慢慢验证 PMF。 
 这套框架已经过时了。时间线被极度压缩， Lean AI 公司跑一个月产生的数据量，相当于传统 SaaS 跑半年到一年。 
 新的评估方式更像评估 ToC 产品那样：紧盯次日留存、7 日留存、30 日留存，以及日活与月活的转化效率（DAU/MAU 比例） 
 如果一款产品在用户注册 30 天后还能留住 30% 的人，已经非常出色。 而 DAU/MAU 一旦超过 50%，基本可以直接做出投资决策 。 
 净收入留存率（Net Dollar Retention）也是一项关键指标。随着越来越多的 Lean AI 公司采用积分或 Token 消耗量计费模式，这一指标的含金量迅速上升。 超过 120% 的净收入留存率才算是一个优秀的信号。 
 这里有一个与大多数 ToC 或游戏公司类似的商业逻辑：少数高付费的"巨鲸"用户（高净值/高频用户）会补贴那些活跃度较低或流失的客群，从而把整体净收入留存率拉上去。 所以在 Lean AI 领域，这个指标远比单纯关注企业客户留存的数量更为重要。

 简单说，这套方法就是把消费科技领域的留存指标打个五六折，套用到 Prosumer（介于 C 端和 B 端之间的）AI 公司身上。 因为对于增长极快的 Lean AI 公司来说，这些指标比传统 SaaS 指标更能反映真实的业务健康度。 
 这恰恰是传统 VC 目前不具备的评估框架。当他们还在等 12 个月的数据时，Lean AI 公司 3 个月已经做到了 3,000 万美元营收。等到准备好出手的时候，机会早已不在了。 
 04. 跑通新模式的基金经济学 
 传统 VC 的基金结构和 Lean AI 公司之间存在一种天然的错配。 
 一只典型的传统 VC 基金规模在 2 到 4 亿美元之间，投 30 到 40 家公司，单笔投资额在 500 万到 1,000 万美元，还要预留 2 到 3 倍的后续跟投储备。这套模型从一开始就是为需要烧钱换增长的资本密集型 SaaS 公司设计的。 
 但 Lean AI 公司的种子轮根本不需要 1,000 万美元。50 万到 200 万美元，就够了。 
 这种错配导致双方的激励机制都出了问题：创始人被迫拿了超出实际需要的钱，VC 则被推着去鼓励烧钱式增长。结果谁也没赢—— 创始人白白稀释了股权，VC 也无法从小额投资中获得理想回报。 
 Yash 的基金结构走了一条完全不同的路： 
 基金规模： 1 亿美元（而非动辄 2 到 3 亿） 
 组合容量： 20 到 25 家公司 
 初始单笔： 种子轮 50 万到 200 万美元，A 轮 500 万到 1,500 万美元 
 预留资金比例： 1 到 1.2 倍（而非传统的 2 到 3 倍） 
 目标持股： 种子轮 15% 到 20%，A 轮灵活处理 
 这套经济模型之所以跑得通，是因为它从一开始就是为资金效率极高的 AI 公司量身设计的，而不是把传统 SaaS 时代的模具硬套上去。 
 但大型平台基金在结构上很难做到这一点。GP（普通合伙人）和 LP（有限合伙人）之间的授权条款限制了灵活性，基金的经济模型也撑不住更小的单笔投资额。部分优秀的小型基金或许能适应，但绝大多数不行——它们被锁死在 2010 到 2020 年间有效的结构里，动弹不得。 
 而这种结构性锁定，恰恰成了 Yash 的竞争优势。 
 05. 预判赢家的三重护城河框架 
 并非所有 Lean AI 公司都值得投。 
 在 Yash 的筛选体系中，有一套三重护城河框架，专门用来区分那些有可能做到十亿美元级别的公司，和注定平庸的公司。 
 护城河一：数据反馈闭环 
 用户在使用产品的过程中会不断产生专有数据，而这些数据反过来又能持续改善产品体验，由此形成一个自我强化的飞轮。这个飞轮一旦转起来，竞争对手即使投入 100 倍的资源也几乎无法复制。 
 这里的关键词是"专有"。通用数据远远不够。在 AI 模型日益同质化的今天， 独有数据才是最具防御力的护城河 。OpenAI 可以用整个互联网来训练模型，但它拿不到每天在某个特定产品中流转的专有数据。 
 护城河二：分发优势 
 独特的 GTM 策略能够触达特定的客户群体，而网络效应、社区和非传统渠道则能构建起大玩家难以复制的壁垒。 
 硅谷很多人还没有意识到一件事 ： 在 AI 时代，分发能力往往比技术本身更具防御力。 OpenAI 可以一夜之间造出更好的模型，但它复制不了一家公司在 某个垂直领域花数月甚至数年建立起来的分发网络 ，也复制不了这家公司在利基市场中积累的社区和品牌影响力。 
 分发本身就是盔甲。 
 护城河三：产品迭代速度 
 速度本身也是一种竞争优势。 
 当一家公司同时拥有专有数据和独特的分发渠道时，它的迭代速度自然会快过所有人。随着数据越多、分发越广、迭代就越快，三重护城河之间会形成指数级的复利效应。 
 更多用户 → 更多数据 → 更快迭代 → 更好产品 → 更多用户。飞轮越转越快。 
 一家公司如果同时拥有这三重护城河，即使 OpenAI 或 Anthropic 进入同一赛道，也几乎不可能被取代。 
 06. Higgsfield：三重护城河的实战验证 
 Higgsfield 是这套三重护城河框架的一个实战案例。 
 Higgsfield 是一家用 AI 生成带有照片级特效短视频的公司，社交媒体内容创作者是它最早的一批用户。 （Linkloud Talk 还曾对创始人做过专访，分享了很多背后的故事。） 如今 Higgsfield 正在切入广告生态，而三重护城河框架在这家公司身上得到了完整的验证。 
 数据反馈闭环： 
 过去一年，Higgsfield 从大量内容创作者的使用行为中积累了海量数据——哪些视频跑得好、哪些特效最受欢迎、哪些模型表现最佳。随着业务切入广告领域，Higgsfield 还能拿到归因数据和 ROI 数据： 用户点击了 Higgsfield 制作的广告之后，最终有没有产生实际购买 。 
 这意味着 Higgsfield 能够精确定位哪个模型、配合什么特效和 Prompt，转化率可以比其他方案高出 10 倍。这是专属于 Higgsfield 的数据护城河，OpenAI 和 Anthropic 都拿不到。 
 分发优势： 
 Higgsfield 与多家模型供应商（Sora 2、Flux、Google Veo 3 等）达成了首发合作，同时在 Reddit 和 Discord 上建立了活跃的社区，依靠有机传播起量。如今它既拥有与内容创作者的直接关系，又打通了通往品牌广告主的通道。这种分发网络是无法在短时间内被复制的。 
 产品迭代速度： 
 专有数据告诉 Higgsfield 什么有效，分发渠道持续为它带来新用户，两者叠加使得 Higgsfield 的产品迭代速度超过了任何竞争对手。 
 更多的试错机会，更多的实验，更多的学习循环。 
 三重护城河在 Higgsfield 身上形成了复利效应。一家公司如果拥有其中一到两个护城河，已经值得关注；三个都有，则可以直接做出投资决策。 
 07. 亦敌亦友：当 OpenAI 进入同一赛道 
 太多 AI 公司在 OpenAI 或 Anthropic 发布新产品之后直接死掉了。这件事怎么避免？ 
 答案是把这些大模型公司当作"亦敌亦友"的关系。 
 要 搞清楚它们的优先级在哪里 。这些大公司迫切需要与前沿 AI 创业公司合作，借此触达自己覆盖不到的垂直领域。所以，如果能 在某个未被充分服务的垂直赛道里率先与它们建立合作关系，价值是非常大的。 
 但战略上必须保持谨慎，尤其在数据共享上要格外警觉：自己的护城河建成之前，不能让大模型公司直接和自己形成竞争关系。可以分享足够多的信息来创造合作价值，但必须保护好那些构成核心防御力的专有数据。 
 这种焦虑其实一直都在：亚马逊会不会直接杀进你的电商细分赛道？AWS 会不会自己造 DevOps 工具？Google 会不会进入你的领域？但事实证明，每个品类最终都出现了多个赢家。 
 关键在于，要在巨头到来之前建好三重护城河。做到了，就能和巨头共存，甚至活得更好。因为基础模型归根结底是通用工具，而在特定垂直领域里长期积累的专业知识、分发渠道和数据，是大模型公司不具备的。 
 这才是真正的防御力。 
 08. 为什么现在是最佳窗口期 
 目前，传统 VC 的注意力几乎全部集中在 AI 基础设施上：基础模型、算力、数据中心。数十亿乃至上万亿美元正在涌向 OpenAI 级别的大交易，整个行业都笃信价值会在基础设施层产生。 
 相比之下，应用层却要冷清得多。 
 历史上有过相似的一幕：在云计算时代，AWS 和 Azure 赚了钱，而且赚得不少。但最大的回报最终属于应用层的公司——Salesforce、Workday、ServiceNow。 基础设施层创造了价值，应用层创造了财富。 
 这个模式正在 AI 领域重演。对于早期行动者来说，有三重优势： 
 第一， 率先理解新一代创始人画像 。当其他人还在寻找斯坦福计算机系毕业生时，已经有人在和那些深耕垂直行业、执行力极强的老兵建立关系了。 
 第二， 在竞争加剧之前抢先建立信任 。这些创始人会记住谁是最早相信他们的人，尤其是在传统 VC 集体 Pass 的时候。 
 第三， 在市场还在成形的阶段率先建立评估体系 。目前市场上还不存在 Lean AI 的投资 Playbook，率先入场的人会拥有先发优势。 
 09. 传统 VC 生存指南 
 目前， A 轮融资的交易数量大幅下降了 79% ，而且资金都在向头部的五家明星公司集中。整个 VC 行业的结构已经无法满足 A 轮创业公司的需求，留下了一片巨大的真空地带等待填补。 
 以下是几条给传统 SaaS 投资人的实操建议： 
 第一， 为 Lean AI 公司单独设立投资池 ，不要把它们硬塞进传统业务模型里。Lean AI 公司拥有完全不同的风险回报结构，用老模型去套只会毁掉回报。 
 第二， 放宽 GP/LP 授权条款 ，让基金结构能容纳更小的单笔投资和更低的预留资金比例。如果基金结构上根本改不动，那就别硬挤进来，老老实实做好自己擅长的传统赛道。 
 第三， 根据需要在不同策略之间灵活调配资金 。有些基金做得到，但大部分大型平台基金做不到。 
 核心就一句话：Lean AI 公司需要一套完全不同的方法论。如果还在用传统 SaaS 指标（2 倍增长预期、300 万到 500 万美元的种子轮、2 到 3 倍的预留资金）去评估它们，一定会错失很多好机会，投资的激励系统也会彻底崩溃。 
 现实情况是，大部分传统 VC 很难做出改变。他们固定的利益分配机制和僵化的经济模型，从根本上不允许这样做。 这就是为什么市场会系统性地对整整一个品类产生定价错误——现有的 VC 基础设施根本接不住这些新的商业模式。

 Linkloud 引言 
 硅谷老牌风投机构 Redpoint Ventures 最近发布了 2026 年度市场报告，用硬数据复盘了软件与 AI 市场正在发生的真实变化。 
 如今，AI 头部初创公司的估值达到 ARR 的 61 倍，看似疯狂，但把增速考虑进去之后，每单位增长的价格比二级市场反而便宜 86%。 最近几个月 SaaS 板块的抛售也不是均匀分布的 ，背后究竟是什么原因？所谓 Task agent  在投资人看来撬动的是一个远超传统软件的 6.2 万亿全新市场 。希望这篇报告为大家带来一些启发。Enjoy! 
 更多完整内容，请点击阅读原文。 
 成立于 1999 年的 Redpoint Ventures ，是硅谷少数横跨两代技术浪潮仍持续命中“时代级公司”的老牌VC之一。从互联网时代一路投到AI时代，它的投资版图几乎就是过去二十年软件产业的进化史：从早期参与 Netflix、Juniper Networks，到云计算周期押中 Snowflake、Twilio、Stripe，再到今天持续布局 AI 基础设施与新一代软件公司。 
 在 AI 投资版图中，从模型之上的应用层，到支撑新一代软件形态的基础设施，它几乎在所有关键节点同时下注：既包括医疗场景中快速放量的 Abridge、自动化财务平台 AppZen，也押注了新一代 AI 编程公司 Poolside、AI 原生设计工具 Flora，以及围绕安全与自动化的新兴玩家如 Irregular、Giga 等。 
 以下是合伙人 Logan Bartlett 联合团队发布的 2026 年度市场报告，通过充分比较后的硬数据，复盘了软件与 AI 最近一年激烈“碰撞”背后的问题和原因。 
 01. 这不是互联网泡沫 2.0 
 每次市场回调，总有人搬出 1999 年互联网泡沫来做对比。但这次类比站不住脚，而且数据也已经给出了清晰的答案。 
 2000 年泡沫顶峰时期，光纤利用率不到 3%，营收基本为零。而今天，OpenAI 和 Anthropic 各自的 ARR 都已突破 200 亿美元；超过 90% 的新建数据中心产能在破土动工前就已被预订一空；电力、土地、网络互联等物理约束，让投机性的过度建设在结构上比互联网时代困难得多，毕竟当年铺设额外的光纤成本极其低廉。 
 另外，ChatGPT 只用了大约 4 年时间就触达 10 亿月活用户，而同样的时间互联网才摸到 7,000 万。 
 基础设施并没有跑在需求前面，是需求在拉着基础设施往前冲 。 
 02. Agent 成熟度曲线还很早期，但 TAM 想象空间极其庞大 
 直到去年，行业基本还停留在 Copilot 阶段：AI 让人效率更高，各家厂商也多在争夺客户现有的软件预算。这有用，但归根到底还是在同一个盘子里抢钱。 
 现在正在进入 Task Agent 阶段， Agent 能够独立运行几分钟，在一定的人工监督下完成端到端任务 。这才是真正的转折点，因为它开始触及人力支出，而不仅仅是软件支出。 
 美国软件市场规模约为 5,000 亿美元，按 Redpoint 的测算，Task Agent 将把市场空间撑大到 1.2 万亿美元。 
 再往前看，Workflow Agent，也就是能跨系统编排多步骤业务流程、持续运行数小时的 AI 已部分落地，全面铺开只是时间问题。 
 Autonomous Agent（自主运行数天、几乎不需要人工介入）离大规模商用还有距离，但已不再是科幻概念。如果真正实现全自主运行，它所能接管的业务范围将远超传统软件的边界，仅美国的知识工作市场就有 6.2 万亿美元 的规模。 
 Agent 的成熟度曲线是真实存在的，而我们还处在非常早期的位置 。 
 03. 软件板块的抛售不是均匀分布的 
 软件板块年初至今下跌 20%，是标普 500 中表现最差的板块。二级市场 SaaS 公司的估值倍数已降至未来 12 个月（NTM）营收的 4.1 倍，创下十年新低。 
 市场对许多此类公司的长期增长预期已趋近于零。但这轮抛售在垂直 SaaS、基础设施和通用 SaaS 三个层面的冲击截然不同，这些差异背后的逻辑值得深究。 
 垂直 SaaS 基本稳住了（过去十二个月上涨 3%）。 
 这类公司往往握有不可替代的护城河：多年积累的行业专属数据、嵌入监管流程的工作流系统、关乎企业存亡而非仅仅是体验层面的转换成本。 
 AI 可以增强这些系统，但想要取代它们，就得把底层数据重新跑一遍。这不是开发一个新的产品需求那么简单，是需要花费十年沉淀的苦功。 
 基础设施大致持平（过去十二个月上涨 2%）。 
 AI 对这个层面而言是顺风助力，而不是威胁。更多的 AI 部署意味着更多算力、更多数据、更多可观测性支出的增加。 
 无论是 Snowflake、Datadog、MongoDB，还是 CrowdStrike、Cloudflare：新的 AI 工作负载全部运行在这些平台之上或经由它们传输。温和的估值压缩更多反映的是利率正常化和一些偏离事实的市场情绪，而非关乎这些企业生死的根本性问题。 
 通用 SaaS 跌了 35%（过去十二个月）。 
 这里的结构性隐忧是真实的。通用应用当初的设计目标是服务所有行业，但实际效果是跟哪个行业都没有深度绑定。 
 它们不掌握行业专属数据，往往也不拥有核心工作流。而且，很多通用 SaaS 产品的核心价值就是回答一个简单问题： 谁在做什么，什么时候做？这本质上是一个协调问题 。而 AI 天生就是解决协调问题的。 
 当产品的核心价值恰好是 AI 的默认能力时，它所面临的威胁性质就完全不一样了 。 
 04. 针对 CIO 的调研数据印证了这一趋势 
 45% 的 AI 预算来自现有软件预算的挪用 而不是净增的新增支出，54% 的 CIO 正在积极推进供应商整合，仅有 3% 的人预期 AI 会带来更多供应商。软件企业现有收入的粘性比表面看起来要弱得多。 
 最容易被 AI 替代的品类也不是随机分布的：销售自动化（83%）、客户服务管理（56%）、IT 服务管理（55%），在这些领域，AI 能够直接下场干活，而不仅仅是充当助手。 
 话说回来，很多老牌企业仍然有机会率先构建 AI 工作流系统。 54% 的 CIO 依然倾向于选择增加了 AI 功能的现有供应商，而不是转向全新的 AI-Native 替代品 。 但真正的考验在于，他们能否把这件事做成。 
 05. "二次创业"真的很难 
 老牌企业面临的挑战不是上线一个 AI 功能，而是打造一家 AI-Native 公司几乎在每一个组织维度上都是不同的。 
 有两点尤为突出： 
 第一，AI-Native 公司不需要那种"见过世面、什么都经历过"的老将，团队需要的是基于第一性原理思考的人才，因为在 AI 这一层，确确实实没有前人的作业可抄。 
 第二，产品开发的逻辑被反转了，传统模式是倾听客户需求然后按需求文档开发，而 AI-Native 团队必须深入理解客户的日常工作场景， 然后从模型的真正具备的能力出发正向推演 ，这要求一种完全不同的产品思维。 
 这不是发布一个移动端 App 那么简单，这是一次深刻的"二次创业"。 如果企业仅仅把它们当做渐进式的功能迭代，那么它们注定会输给那些将其视作架构重置的竞争对手 。 
 06. 一级市场估值：溢价看似离谱，算上增速就不离谱了 
 Redpoint 追踪的头部高增长 B 轮和 C 轮软件公司，2026 年至今的融资中位数估值为 ARR 的 61 倍，而二级市场高增长软件公司的交易倍数为 9.7 倍，528% 的溢价听起来好像与现实脱节。 
 但这批一级市场公司的 ARR 增长率中位数高达 640%。相比之下，二级市场高增长软件的增速约为 29%。 如果用估值倍数除以各自的增长率来衡量性价比，一级市场公司反而比二级市场便宜了 86% 。 也就是说，一级市场虽然估值倍数高，但增速也高得多，算下来每一分钱买到的增长空间反而比二级市场大得多。 
 当然，这种分析并不十分严谨。两个市场不能简单做对标比较，而且这种规模的增速也很难长期维持。但考虑到除少数几家巨头外，二级市场整体的增长空间已被极度压缩，单纯比对估值倍数会掩盖事物运转的真实面貌。 
 07. 如果历史具有借鉴意义，那么我们正处于时间窗口之中 
 回顾互联网、云计算和移动互联网三次平台级变革，最终成为长期赢家的公司大多诞生在每次浪潮的第 4 到第 5 年。 
 互联网时代，Netscape 之后跑出了 Google 和 Salesforce。云计算时代，AWS 发布后诞生了 Snowflake 和 Datadog。移动互联网时代，App Store 之后崛起了 Robinhood 和 Coinbase。 
 ChatGPT 于 2022 年 11 月发布，现在是第 4 年。OpenAI 和 Anthropic 确实有可能在这一轮吃下最大份额的价值，但过去三次平台变革的规律在这一点上高度一致。 
 当下也是 Logan 从业以来见过的竞争最激烈、节奏最快、门槛最高的创业环境，融资轮次在几天内关闭，估值从第一天起就要求极致的执行力来兑现。每天都兴奋，也每天都不确定。
