# AIGC视觉生成

## 本周进展与趋势

本周AIGC视觉生成领域呈现出三大核心突破：**移动端实时化、视频生成精细化与3D视觉世界建模升维**。在移动端部署方面，字节开源的0.39B参数模型实现了手机本地实时图像生成与编辑，大幅降低了应用门槛。在视频生成方面，小米SVOR框架通过稳定视频对象移除技术攻克了阴影残留与运动抖动难题，并获得CVPR 2026冠军；同时，HeyGen揭示了后训练数据中token退化（如日语退化率高达29.7%）是导致模型输出错误语言字符的根因，为模型微调提供了关键洞见。在3D视觉方向，CVPR 2026涌现出E-RayZer（无需3D标注或相机位姿的自监督重建）、LagerNVS（3D-aware特征实时新视角合成）等突破，标志着模型正从“二维感知”向“三维世界建模”跃迁。此外，何恺明团队通过iMF等生成范式优化，将单步生成FID提升至1.72；美图RoboNeo首创Agent Teams协作架构，进一步推动了多智能体影像创作流程的自动化与创意多样性。

### 关键趋势
- **趋势1：轻量化模型推动AIGC移动端落地**
- **趋势2：视频生成中的细节鲁棒性与语言退化问题被深入攻克**
- **趋势3：3D视觉从二维理解升维至“世界建模”，涌现无监督与新视角合成技术**
- **趋势4：生成范式优化与多智能体协作提升创作效率**

# 多模态大模型/大语言模型

## 本周进展与趋势

本周多模态与大语言模型领域呈现出三大核心进展：**语音交互能力跃升、模型可解释性与安全对齐深化、以及推理范式转向长程与鲁棒性。** 在语音方向，OpenAI发布GPT-Realtime-2系列，实现128K上下文、并行工具调用及五档可调推理强度，在Big Bench Audio上提升15.2%；阶跃StepAudio 2.5则在Artificial Analysis盲测中位列中国第一、全球前三，并支持千万级人设精细化控制。与此同时，Anthropic提出自然语言自动编码器（NLA），将模型隐藏动机发现率从<3%提升至12-15%，为预部署对齐审计提供了新工具。推理研究方面，DeepSeek多模态报告揭示“指代鸿沟”是推理失败主因，并创新引入空间标记基元；Meta/斯坦福的ProgramBench测试显示所有模型在复杂软件重编程任务上完整通过率为0%，而Next-ToBE方法则通过软目标分布激活模型前瞻能力，试图打破“自信而短视”的NTP训练诅咒。此外，关于token退化机制的分析（如“马嘉祺”消失）为模型输出遗忘提供了解释，而Heuristic Learning框架则提出一种超越梯度训练的范式可能。

### 关键趋势
- **语音AI进入“可调智能”与“深度定制”阶段**：OpenAI GPT-Realtime-2支持五级推理强度（minimal至xhigh），在反应速度与推理深度间提供开发者控制；阶跃StepAudio 2.5 TTS基于RLHF对齐优化，支持基于10,000+人设和百万级特征矩阵的精细调节，赋予语音模型极高的角色扮演与语境响应能力。
- **模型可解释性与安全对齐的工程化突破**：Anthropic的NLA方法将模型内部高维激活值翻译为自然语言解释，并具备逆向重建验证能力，已用于Claude Opus 4.6的实际对齐审计，将隐藏动机发现率从<3%提升至12-15%，超越了仅依赖思维链（CoT）的监控方式。
- **推理评估转向“零参考”与“复杂长程”挑战**：ProgramBench测试（200个任务，含FFmpeg、PHP等8,635行中位数代码）显示，所有模型在仅凭可执行文件复现行为时完整通过率达0%，其中Claude Opus 4.7以51.2%平均测试通过率领先。同时，DeepSeek多模态工作将“指代鸿沟”作为推理失败主因，并以81个视觉KV entries压缩实现迷宫/路径追踪任务超越GPT/Claude。
- **训练范式革新：软目标学习与规则智能蒸馏**：Next-ToBE（ICLR'26）提出用软目标分布替代one-hot分布，量化并优化模型对长程token的覆盖能力（FtHR指标），不改变模型结构即可提升多步推理性能；OpenAI翁家翌提出的Heuristic Learning框架则利用coding agent自动迭代手写规则系统，配合回归测试与seed回放规避遗忘，在游戏和机器人控制中达到媲美Deep RL的性能。

本周自动驾驶与具身智能领域呈现显著的技术路线分化与聚焦。一方面，以英伟达Jim Fan为代表的观点认为当前主流的VLA（视觉-语言-动作）范式正被更底层、更通用的WAM（世界动作模型）取代，其代表作DreamZero及基于海量人类视频（如21,000小时、44,000小时）训练的神经缩放定律与神经仿真器，预示着到2040年实现物理自主研究的终局路径。另一方面，产业界和学术界依然在VLA路线上快速迭代：高德ABot模型在AGIBot挑战赛中以0.829总成绩夺冠，印证了具身空间智能结合高精度地图的有效性；港科大开源的StarVLA框架整合了RT-2、π₀等主流模型，并提供了可插拔模块设计，成为国内热度最高的VLA工具之一。与此同时，关于数据与基础设施的争论愈发激烈——Ted Xiao复盘指出模仿学习路线已超越强化学习，而简智机器人则明确放弃模型竞赛，专注从头到手的多模态高精度数据采集与数据闭环基建。整体来看，行业正从模型架构的单一竞争，转向“世界模型、数据基建、任务泛化”三大维度的系统性突破。

### 关键趋势

# Coding Agent

## 本周进展与趋势

本周（共23篇文章）围绕Coding Agent的安全、记忆、基础设施与开发范式展开密集讨论。安全领域成为焦点：多篇文章指出Agent自主执行代码带来的删库风险与数据泄露问题，并提出了从操作系统层零代码感知（清华Fangcun Observer）、运行时治理内核（港中文ArbiterOS，高危拦截率达92.95%）到安全沙箱与审批工作流（OpenAI Codex四层架构）的全链路解决方案。记忆系统呈现结构化与持久化趋势，Hermes Agent的四层记忆系统、魔搭Ultron的Memory Hub（三级热度管理）以及Cloudflare推出的Agent Memory托管服务（支持双通道摄入与五通道检索）均试图解决上下文衰减与跨会话知识迁移问题。同时，AI编程的“非确定性”特性对分布式基础设施提出新挑战，需要更安全的容器/虚拟机与适配K8s的调度方案。此外，“非程序员用SOP封装技能”与“AI软件工厂驱动组织Agent”等现象，标志着编程正从“写代码”转向“定义流程、审查与交付”，而“认知债务”与初级开发者培养渠道萎缩的警告则要求业界重视系统性验证能力。

### 关键趋势
- **安全治理从被动防御走向全链路运行时拦截**：多个团队在操作系统层或Agent框架内部署实时监控与拦截机制。清华团队提出的Fangcun Observer能在零代码侵入下感知命令、文件与网络操作，Guard模块延迟<100ms；港中文ArbiterOS将高危步骤拦截率从6.17%提升至92.95%。这些系统还支持动态贝叶斯网络、可微攻击算法及运行时行为与模型上下文的关联溯源，推动安全能力从静态规则进化为本地自进化防御。
- **记忆系统从扁平缓存进化为结构化、跨会话的持久化基础设施**：Hermes Agent采用冻结Prompt+SQLite归档+Skills的四层设计，通过字符限制和Prompt前缀缓存优化实现模型无关记忆。魔搭Ultron的Memory Hub按HOT/WARM/COLD三级热度管理，基于使用反馈动态计算权重。Cloudflare的Agent Memory则提供事实、事件、指令、任务四类记忆，支持五通道并行检索（RRF融合）与团队共享档案，依托边缘分布式部署实现低延迟。
- **Agent非确定性催生新型分布式基础设施需求**：传统确定性基础设施无法匹配Agent动态逻辑导致的资源预估难题与安全挑战。文章指出，AI生成代码和外部工具不可信，传统容器隔离性不足，需采用更安全的沙箱或虚拟机并实现与K8s等框架的深度对接。这成为制约Agent大规模落地的核心瓶颈，同时也为基础设施领域带来创新机会。
- **开发范式从“写代码”转向“审查与封装”，组织Agent模式兴起**：Claude Code之父Boris Cherny称2026年尚未亲手写一行代码，预测未来十年创业公司数量增加10倍；GStack项目（获9万+Star）定位为“AI软件工厂”，用Markdown驱动23个专业角色的虚拟团队。同时，“Skill”概念开始出圈——非程序员用SOP封装出可迭代的“经验账户”，底层模型成熟后，流程定义和系统性验证成为最稀缺能力。

## [Hermes Agent 的记忆系统：为什么它修正了 OpenClaw 的错误](https://mp.weixin.qq.com/s?__biz=MzU2NzkxNDY0Ng==&mid=2247490336&idx=1&sn=073826c9f1a2ee6b14437c6defa31a6b)

**来源:** 山行AI

### 摘要
本文深入解析了开源AI Agent Hermes的记忆系统，指出其为解决OpenClaw等Agent的记忆问题，设计了四层记忆架构：1) 冻结的Prompt记忆，包含MEMORY.md和USER.md两个文件，分别限制2200和1375字符，用于存储高价值的Agent笔记和用户画像，在会话中冻结快照以优化缓存；2) 可搜索的SQLite历史会话归档；3) Agent管理的Skills作为程序性记忆；4) 可选的Honcho层用于深层用户建模。核心设计理念是保持Prompt稳定以利用缓存，将其他信息通过工具按需检索。记忆以纯文本、字符限制（而非token）的方式存储，与模型无关。系统Prompt的组装顺序也体现了对缓存优化的考虑。总结认为Hermes将记忆视为经过筛选的状态而非日记，这是其与OpenClaw等系统的关键区别。

### 关键要点
- Hermes拥有四层记忆系统：冻结Prompt、SQLite归档、Skills、Honcho
- MEMORY.md和USER.md分别限制2200和1375字符，以纯文本和字符限制实现模型无关记忆
- 关键设计：保持Prompt前缀稳定以优化缓存，将其他信息移出Prompt按需检索
- 记忆在会话中冻结为快照，写入立即落盘但仅在新会话或触发重建时更新Prompt

Cloudflare在Agents Week期间宣布推出Agent Memory托管服务的私人测试。该服务旨在解决AI智能体在多会话中的“上下文衰减”问题，提供跨会话的持久化记忆能力。核心机制是，无需将全部内容塞入上下文窗口，而是从对话中提取结构化记忆，仅按需检索。技术架构为双通道摄入（宽泛通道处理约10K字符分块，细节通道提取具体数值），经八项校验将记忆分为事实、事件、指令、任务四类。在检索端，通过五条并行通道（全文搜索、精确键查找、原始消息搜索、向量搜索、HyDE补充）并使用倒数排名融合（RRF）整合结果。默认使用Llama 4 Scout (17B MoE)提取信息，仅合成时使用更大的Nemotron 3 (120B MoE)。特色功能是共享记忆，允许团队共用知识。差异化优势在于边缘分布式部署、与Cloudflare自身原语深度集成。目前处于私人测试阶段。

### 关键要点
- 问题：Agent面临上下文衰减（Context Rot），长上下文导致模型输出质量下降。
- 方案：Agent Memory托管服务，提供跨会话、持久化、结构化记忆能力。
- 技术：双通道摄入、四类记忆（事实/事件/指令/任务）、五通道并行检索（RRF融合）。
- 模型：默认使用Llama 4 Scout (17B MoE)提取，仅合成时使用Nemotron 3 (120B MoE)。
- 特色：支持团队共享记忆档案，实现知识迁移；差异化在于边缘分布式部署与深度集成。
